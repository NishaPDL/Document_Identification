🏢 HRMS Employee Document Upload Integration

📋 Integration Instructions

This example shows how to integrate the document classification service with your HRMS Employee page at https://predemoui.paisalo.in:4022/hrms/Employee

⚙️ Configuration

Update the configuration in the JavaScript file to match your setup:

const config = { apiBaseUrl: 'http://your-java-service:8080/api', // Your Java service URL employeeIdField: 'employeeId', // Your employee ID field name debug: true // Set to false in production };

👤 Employee Information Form (Example)

Upload Aadhaar document (PNG, JPG, JPEG, WebP, BMP)

📝 Integration Steps

  1. Include the JavaScript file in your HRMS page:
    <script src="document-upload-integration.js"></script>
  2. Update the configuration to match your Java service URL and employee ID field
  3. Ensure your file input has the correct ID:
    <input type="file" id="employeeaadhar" name="employeeaadhar" accept="image/*">
  4. The integration will automatically:
    • Detect when a file is selected
    • Validate it's an image file
    • Send it to your Java service for classification
    • Verify it's an Aadhaar document
    • Upload it to the organized folder structure
    • Show success/error feedback
    • Store the upload path in a hidden field for form submission

🎯 Expected Behavior

  • ✅ Valid Aadhaar: Document verified and uploaded to /uploads/{employeeId}/aadhaar/
  • ⚠️ Wrong Document Type: Warning shown if document is not classified as Aadhaar
  • ❌ Invalid File: Error shown for non-image files or processing failures
  • 📁 Upload Path Stored: Hidden field aadhaarUploadPath contains the file path for form submission

🔧 API Endpoints Used

  • POST /api/classify-and-upload-single - For single file upload and classification
  • GET /api/upload-stats/{userId} - For getting upload statistics (optional)
